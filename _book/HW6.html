# 1.) Write down the linear regression model 
# mathematically. Use matrix notation Y = Xβ + ε.
# Identify each component: what is Y, what 
# columns are in X, what does each element of β
# represent?

# Y = Xβ + ε
# Y = dependent variable --> response or obsered dependent response variable
# X = independent variable --> Design matrix
# β = coefficient --> (X^T X)^-1 X^T Y
# X^T = transpose of the matrix
# (X^T X)^-1 --> inverse of the matrix product X^T * X
#   * example of an inverse of a matrix: a^-1 = 1/a --> a = [a, b, c, d] and a^-1 = [1/a, 1/b, 1/c, 1/d] or A^-1 = 1/|A| * Adj(A) = 1/(ad-bc) * [d, -b, -c, a]
# ε = error -- > # ε = Y - Xβ = ŷ - y
# ŷ = predicted value of y = Xβ

# Set factor levels
insurance_data$sex    <- factor(insurance_data$sex,    levels = c("female", "male"))
insurance_data$region <- factor(insurance_data$region, levels = c("northeast", "northwest", "southeast", "southwest"))
insurance_data$smoker <- factor(insurance_data$smoker, levels = c("no", "yes"))

# creating design matrices to combine together all of the independent predictor variables of x
sex_matrix    <- model.matrix(~ 0 + sex,    data = insurance_data)

smoker_matrix <- model.matrix(~ 0 + smoker, data = insurance_data)
age_matrix     <- as.matrix(insurance_data$age)
bmi_matrix     <- as.matrix(insurance_data$bmi)
children_matrix <- as.matrix(insurance_data$children)

# what are columns are in X and what does they represent?

# Use generalized inverse for the Coefficient Estimate equation (X^T * X)^-1 * X^T * Y

b <- ginv(t(x) %*% x) %*% t(x) %*% y

# assigning the column and row names

# For all of the columns are in X, what does each element of β represent?

# For each element of β in combination with the columns of x (age, bmi, children, etc...),
# the element of β tells us how much that particular column in X contributes and influences the calculation of the predicted value of Y.
# In this case though, we are using the linear combinations of the columns describing the features of the individuals (age, bmi, children, etc...)
# along with the elements of β (to increase or decrease weight or influence) to calculate the predicted values of y or in this case the charges of health insurance.
# In otherwords, the linear combinations of the columns of x and the elements of β helps determine which features of the individuals has more or less influence on the predicted insurance charges for these individuals.

# What additional output does ols() provide that lm() does not?

# When cross comparing the outputs of ols() and lm(), we can see that ols() provides a number of different
# features/statistical measures that give different statistical aggregated measures about the data that lm() does not have.
# For example, ols() provides features/variables such as Lower 0.95 and Upper 0.95 that shows the lower and upper bonds of a 95% confidence interval
# along with Low and High features/variables that highlights the maximum and minimum values of the data along with a Diff = Difference feature that outlines
# the difference between the maximum and minimum values in the ols() outputs and finally it has a Type feature that highlights the type of the data.
# While the output of lm() only uniquely posseses the t value and Pr(>|t|) features/variables that highlights the p-value of a observation that is higher than the t-test statistic value and
# the actual t-statistic value of the data.
# The only features/variables that these outputs actually shares is the Estimate and Std. Error features/variables along with the Effect and S.E. features/variables that is partically the same features/variables
# which is that they are the Coefficient β and Standard Error features/variables for each model.
